## Git 설치
Git 설치: <https://git-scm.com/install/windows>

![](/assets/img/Git/installation.png){: width="500" .normal }

1. 위 사이트 접속 후 `Git for Windows/x64 Setup.` 클릭
<br>

2. exe 파일 실행 후 다음과 같이 변경

![](/assets/img/Git/setup1.png){: width="500" .normal }

VSCode를 에디터로 사용할 것이기 때문에 다음과 같이 변경
<br>

![](/assets/img/Git/setup2.png){: width="500" .normal }

브랜치 이름을 master 대신 main으로 짓고 있는 사람이 늘어나고 있는 추세여서 main으로 변경
<br>

3. 설치 완료 후 동작 확인

깃이 정상적으로 설치되서 작동하는지 확인해보고 싶으면 cmd 창에 git --version을 입력하면 됨
<br>

1. (처음에만 설정) git 유저 이름 설정해주기  
`git config --global user.email "깃허브 이메일"`  
`git config --global user.name "깃허브 닉네임"`
<br>

### 사용해보기

1. 설치한 VSCode를 열어 상단의 File - Open Folder... 를 통해 관리하고자 하는 폴더 열기
<br>

2. 해당 폴더에서 Git을 사용하고자 하면 상단의 Terminal - New Terminal 선택하여 VSCode의 터미널 창을 띄운 후 `git init` 입력  
  아래와 같이 설정이 완료됨  
  PS C:\Users\wyn\Desktop\git> git init  
  Initialized empty Git repository in C:/Users/wyn/Desktop/git/.git/  
<br>

3. `git add 파일명.확장자` - 기록을 하고자 하는 파일 생성
<br>

4. `git commit -m '변경사항 등 메모할 사항'` - 현재 상태를 기록함  
  아래와 같이 commit이 됨  
  PS C:\Users\wyn\Desktop\git> git commit -m '첫 코드 생성'  
  [main (root-commit) 49edb00] 첫 코드 생성  
   1 file changed, 1 insertion(+)  
   create mode 100644 test.txt  
<br>

### 기초적인 명령어

여러 개를 동시에 스테이징  `git add 파일1 파일2`

작업 폴더를 전부 스테이징  `git add .`

변경된 파일이나 스테이징 파일 알고 싶을 때  `git status`

스테이징 된 파일을 취소하고 싶음  `git restore --staged 파일명.확장자`  `git restore --staged .` (전체 취소)
<br>

### &#8251; 용어
- staging area  
  폴더 안에 다양한 파일들이 있는데 어떤 파일을 add하면 그곳이 staging area가 됨  
  commit 할 파일들을 골라두는 곳  
  스테이징 한다고도 부름  
<br>

- repository (저장소)  
  commit된 파일의 버전들을 모아두는 곳
